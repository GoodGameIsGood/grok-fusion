# Composition Rules

Web composition laws for landings and promotional surfaces. Product app chrome may densify, but still keep one job per region.

## First viewport

- One composition, not a dashboard collage
- Brand/product name at hero-level weight
- Full-bleed hero visual by default (edge-to-edge plane)
- Hero budget only: brand, one headline, one short support line, one CTA group, one dominant image
- No cards in the hero
- No floating badges, chips, or promo stickers on hero media
- No stats strips, schedules, address blocks, or secondary promos in the first viewport

## Sections

- One job per section; one headline; usually one short supporting sentence
- Prefer sections, columns, dividers, and media blocks over card grids
- Cards only when they wrap a real interaction
- Real visual anchor (product/place/context) — not abstract decoration alone

## Hierarchy

- Clear type scale from tokens (display / body / utility)
- Primary CTA obvious; secondary quieter
- Asymmetry (e.g. 60/40) beats forced centered-everything when it serves the brief

## Motion

- Ship 2–3 intentional moments (load, scroll reveal, or hover) on visually led pages
- Prefer orchestration over scattered micro-noise
- Always honor `prefers-reduced-motion`
