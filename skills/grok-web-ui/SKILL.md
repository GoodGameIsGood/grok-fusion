# Grok Web UI

Generative web/UI satellite. Always honor `grok-design` canon and tokens-preflight first.

## Required reads

1. `grok-design` → [design-canon.md](../grok-design/references/design-canon.md)
2. `grok-design` → [tokens-preflight.md](../grok-design/references/tokens-preflight.md)
3. This skill → [references/composition-rules.md](references/composition-rules.md)
4. On demand → [references/style-recipes.md](references/style-recipes.md) (pick at most one)
5. Before ship → [references/responsive-a11y.md](references/responsive-a11y.md)
6. Critique → `grok-design` [screenshot-critique.md](../grok-design/references/screenshot-critique.md)

## Process

1. Lock tokens and archetype (preflight)
2. Structure HTML with clear landmarks and one H1
3. Implement CSS from tokens; expressive type; atmospheric background
4. Add 2–3 intentional motions; respect reduced-motion
5. Self-critique (soft screenshot when available)
6. On Fusion visual waves, expect pack `visual-ui` / specialist `visual_design_critique`

## Brownfield

If the project already has a design system or visual language, **preserve it**. Extend tokens and components; do not invent a parallel aesthetic.
